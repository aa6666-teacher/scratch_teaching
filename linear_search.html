🔍 主題：找出隱藏的資料——線性搜尋法 (Linear Search)

用最直觀的方式，從資料第一筆開始依序比對目標值

💡 學習目標：理解搜尋概念、掌握線性搜尋流程、能用虛擬碼表達，並能進一步用 Scratch 實作。

一、 什麼是「搜尋」？

在日常生活中，我們經常需要從一大堆東西中找出一件特定的物品。例如：在書架上找一本特定的書。在通訊錄 裡找某人的電話。在抽屜裡找一雙成對的襪子。
在電腦科學中，「搜尋」(Search) 就是在一個資料結構（如陣列、列表）中，找出特定目標值的過程。

二、 線性搜尋法 (Linear Search) 的原理

線性搜尋法（又稱循序搜尋法）是最直觀、最簡單的搜尋方式。核心邏輯：從資料的第一筆開始，依序與目標值進行比較， 直到找到目標值為止；或者如果比到了最後一筆都沒找到，就代表資料不在裡面。

🏫 生活模擬：點名時間

想像班上有 30 位同學排成一列，老師想找「王小明」：
老師問第 1 個同學：「你是王小明嗎？」(不是，繼續找下一個)
老師問第 2 個同學：「你是王小明嗎？」(不是，繼續找下一個)
老師問第 3 個同學：「你是王小明嗎？」(喔！找到了！) → 結束搜尋

三、 線性搜尋的步驟（演算法流程）

1 設定目標（Target）：決定要找的數值。
2 從頭開始：從資料清單的第一個元素（索引值 0）開始看。
3 依序判斷：如果目前的數值等於目標值，就「找到」並結束；否則往後移一格，看下一個數值。
4 檢查結束：如果已經看到最後一個元素都沒找到，代表目標值不在這份資料中。

四、 線性搜尋的優缺點

優點

  • 簡單易懂：邏輯非常直覺。
  • 不限順序：資料不需要先經過排序，亂序也可以找。
  • 最好情況快速：第 1 筆就找到。

缺點

  • 效率較低：資料量很大時會變慢。
  • 最壞情況要比對 $n$ 次。

五、 動動腦：作業：用 Scratch 實作線性搜尋

在 Scratch 中，我們通常會使用「清單 (List)」來儲存資料，並用「變數」來記錄我們找的進度。

1. 準備工作（變數與清單）

  • 清單 資料庫：存放我們要搜尋的所有數字。
  • 變數 目標：我們要找的那個數字。
  • 變數 索引 (Index)：記錄目前查到第幾筆（從 1 開始）。
  • 變數 是否找到：預設為「否」。

2. Scratch 積木邏輯圖解

  • 請用 Scratch 截圖/畫面：畫出「初始化」到「重複執行」到「找到/沒找到輸出」整段流程。
  • 重點積木要包含：綠旗被點擊、詢問並等待、重複執行直到（或等價迴圈）、如果 與 停止/說出。

3. 核心程式邏輯（虛擬積木描述）

當綠旗被點擊時：

  • 詢問「你想找哪個數字？」並將結果設為 目標。
  • 將 索引 設為 1。
  • 將 是否找到 設為 false（「否」）。

重複執行：直到 索引 > 資料庫 的長度 或 是否找到 = true。

  • 如果 資料庫 的第 索引 項 = 目標： 說出「找到了！在第」 + 索引 + 「項」持續 2 秒，並將 是否找到 設為 true。
  • 否則：將 索引 改變 1（往下一筆看）。

重複結束後：

  • 如果 是否找到 = false：說出「抱歉，資料庫裡沒有這個數字」持續 2 秒。
💡 Scratch 實作小撇步（學生常錯點）
  • 索引的起始值：在 Python 中索引通常從 0 開始，但在 Scratch 清單中是從 1 開始，這點要特別提醒同學。
  • 停止搜尋：找到目標後，一定要記得讓 是否找到 變成 true 或使用「停止這個腳本」，否則程式會繼續笨笨地把剩下的資料比對完。
