演算法入門：插入排序

國中二年級 · 圖解說明 · 例題 · Scratch 實作引導

相關講義： 線性搜尋　· 二元搜尋　· 選擇排序。 本篇是第二種排序法，目的同樣是「把亂的清單排成由小到大」，但和選擇排序的做法完全不同。

本課要學什麼？

  • 理解插入排序的核心：每次拿起一個元素，往左比較、右移、再插入正確位置。
  • 能說出「右移」和「插入」各是什麼意思，以及為什麼需要暫存 temp。
  • 能用表格一輪一輪模擬比較和移位的過程。
  • 能說出插入排序在「資料幾乎已排好」時，為什麼比選擇排序更快。
  • 能在 Scratch 裡用清單＋雙層迴圈實作插入排序。
一、靈感來源：整理手牌

打撲克牌時你怎麼整理？

玩撲克牌時，每摸到一張新牌，你不會把手上所有牌重新洗一遍——而是把新牌插入已排好的牌裡：

  1. 把新牌和手上最右邊（最大）的牌比一比。
  2. 如果新牌比較小，就把那張牌往右移一格，再繼續往左比。
  3. 直到找到一張牌比新牌還小（或已到最左邊），就把新牌插進去。

這就是插入排序（Insertion Sort）的核心——每次從「還沒整理的部分」拿起第一個，找到它在「已整理部分」的正確位置，然後插進去。

一句話記住：拿起新牌 → 往左一張一張比 → 比我大就右移 → 找到位置就插入。已排好的部分每輪多一個，最後全部整理完畢。

和選擇排序的最大差異

兩種排序，兩種策略

  • 選擇排序：每輪先把整個未排序區掃完，找出最小值，再和最前面交換。每輪最多一次交換，但一定要掃完整個未排序區。
  • 插入排序：每輪直接拿起下一個元素，往左一個個比，比我大的就往右移，找到位置就立刻停。資料越接近排好，可以更早停下來，非常有效率。
二、四個關鍵動作

插入排序每一輪有四個動作，依序執行：

每輪的四步驟
  1. 拿起（暫存）：把當前位置 i 的值存進 temp（因為等一下這格可能被右移的值蓋掉）。
  2. 設定起點：把比較指標 j 設為 i－1（從 temp 的左邊一格開始）。
  3. 往左比、右移：若「第 j 項」比 temp 大，就把第 j 項複製到第 j＋1 項（右移），然後 j 往左移一格（j－1）；重複直到 j < 1 或第 j 項 ≤ temp。
  4. 插入：把 temp 放入第 j＋1 項（剛空出來的位置）。
為什麼要用 temp？右移時會用「第 j 項」覆蓋「第 j＋1 項」。若不先把第 i 項的值另存，右移一展開，那個值就消失了。

「右移」和「交換」有什麼不同？

插入排序的移位是單向覆蓋

執行 第 j＋1 項 ← 第 j 項 時，只是把第 j 項的值複製到右邊一格，不需要再用額外的 temp 來「換回去」——因為原本第 j＋1 項的值，要么已經存在 temp 裡（第一輪），要么已經在上一次右移時被往右搬走了。

最後只要把 temp 放到空出來的第 j＋1 項，就完成了。

三、圖解：一輪一輪走一遍

以清單 [4, 2, 7, 1]（共 4 個元素）為例，目標排成 [1, 2, 4, 7]。

顏色說明： ■ 綠色 = 已排序（確定）； ■ 橘色 = 當前 key（存入 temp）； ■ 藍色 = 未排序。

第 1 輪（i = 2）

temp = 2。j = 1，第 1 項 4 > 2 → 4 右移；j = 0，超出左界 → 插入 2 到第 1 項。

第 2 輪（i = 3）

temp = 7。j = 2，第 2 項 4 < 7 → 立刻停止，不需要右移，7 插回原位。

第 2 輪只比了一次就停——這是插入排序的優點，遇到已在正確位置的元素可以立刻結束。

第 3 輪（i = 4）

temp = 1。往左比：7 > 1 右移、4 > 1 右移、2 > 1 右移，j = 0 超出左界 → 插入 1 到第 1 項。

本輪內層迴圈逐步過程：

—temp = 第 4 項 = 1存入 temp[2, 4, 7, temp=1]
j = 3第 3 項 7 > 1第 4 項 ← 7（右移）[2, 4, 7, 7]

4 個元素跑了 3 輪（n－1 輪）。第 2 輪一次比較就停；第 3 輪移了 3 次。

四、流程步驟與比較次數
插入排序流程（由小到大）

清單長度為 n，位置編號 1 到 n。

  1. 讓 i 從 2 跑到 n（共 n－1 輪）：
  2. 　2-1. temp ← 清單第 i 項（拿起 key）。
  3. 　2-2. j ← i－1。
  4. 　2-3. 當 j ≥ 1 且 清單第 j 項 > temp：
    　　　清單第 j＋1 項 ← 清單第 j 項（右移）；j ← j－1。
  5. 　2-4. 清單第 j＋1 項 ← temp（插入）。

比較次數：最好 vs 最壞

情況說明每輪比較合計（n 個元素）
最好情況 資料已由小到大排好 每輪只比 1 次（立刻停） n－1 次
最壞情況 資料由大到小（完全倒序） 第 i 輪比 i－1 次 n×(n－1)÷2 次
比選擇排序更靈活：選擇排序不管原始順序如何，比較次數永遠是 n×(n－1)÷2。插入排序在最好情況下只需 n－1 次——資料越接近排好，它越快！

移動（右移）次數

每次比較若 key 較小就移動一次，所以移動次數和比較次數相同（或差 1）。最好情況移動 0 次；最壞情況移動次數同樣為 n×(n－1)÷2 次。

五、超短虛擬碼（對照 Scratch）
i 從 2 到 清單長度： temp ← 清單第 i 項 （拿起 key） j ← i - 1 當 j ≥ 1 且 「清單第 j 項」 > temp： 清單第 j+1 項 ← 清單第 j 項 （右移） j ← j - 1 清單第 j+1 項 ← temp （插入）
Scratch 的「重複直到」是停止條件，要反過來寫：繼續條件是「j ≥ 1 且 第 j 項 > temp」，停止條件就是相反：j < 1 或 NOT（第 j 項 > temp）。在 Scratch 裡用「或」和「not」積木組合。
索引技巧：Scratch 清單的索引不能直接用算式，先把 j＋1 的結果存進一個 j1 變數，再用 j1 當索引，就能正確替換第 j＋1 項。

七、Scratch 實作引導

準備

  • 建立清單（例如命名為「資料」），手動輸入數字。
  • 準備變數：i（外層迴圈）、j（往左比較的指標）、j1（= j＋1，清單索引用）、temp（暫存 key）。
  • 選做：加「比較次數」和「移動次數」變數，驗算最好／最壞情況是否符合公式。

積木怎麼接（概念）

  1. 外層迴圈：讓 i 從 2 跑到「資料長度」。
  2. 外層迴圈開始時：temp ← 資料第 i 項，j ← i－1。
  3. 內層「重複直到」條件：(j < 1) 或 NOT (資料第 j 項 > temp)。
    • 迴圈內：先 j1 ← j＋1，執行「替換資料第 j1 項為資料第 j 項」（右移）；再 j ← j－1。
  4. 內層迴圈結束後：j1 ← j＋1，執行「替換資料第 j1 項為 temp」（插入）。
除錯小技巧：每輪外層迴圈結束後讓角色說出 temp 和當下清單狀態，確認 key 插入了正確位置。若發現清單某項消失，通常是右移時索引算錯，檢查 j1 的值。

四種演算法總整理

演算法 目的 需先排序？ 最好比較次數 最壞比較次數
線性搜尋 找目標 不需要 1 次 n 次
二元搜尋 快速找目標 要 1 次 約 log₂n 次
選擇排序 排序 — n×(n－1)÷2 n×(n－1)÷2
插入排序 排序 — n－1 次 n×(n－1)÷2
組合應用：先用插入排序或選擇排序把資料整理好，再套用二元搜尋快速查找，這是把多種演算法串在一起的完整流程！
八、延伸思考（選做）
  • 若想改成「由大到小」排序，流程裡哪一個比較符號要改？改成什麼？
  • 清單 [1, 3, 5, 2, 4] 用插入排序，總共比幾次、移動幾次？試著模擬看看。
