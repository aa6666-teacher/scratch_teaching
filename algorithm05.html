演算法入門：氣泡排序

國中二年級 · 圖解說明 · 例題 · Scratch 實作引導

相關講義： 線性搜尋　· 二元搜尋　· 選擇排序　· 插入排序。 本篇是第三種排序法，也是最有「畫面感」的一種——元素像氣泡一樣，一步一步往右浮到正確位置。

本課要學什麼？

  • 理解氣泡排序的核心：每次比較相鄰兩個元素，若左大右小就互換，反覆進行直到排好。
  • 能說出為什麼每輪結束後，最大值一定會「浮到」最右邊。
  • 能用表格模擬每一輪每一次的比較和交換過程。
  • 理解「提前停止」最佳化：一輪內完全沒有交換，就代表已排好，可以立刻結束。
  • 能在 Scratch 裡用清單＋雙層迴圈＋旗標變數實作氣泡排序。
一、為什麼叫「氣泡」排序？

氣泡往上浮的比喻

想像一杯水裡有許多大小不一的氣泡，較大的氣泡因為浮力更大，每次都會比小氣泡先往上浮一格，最後「最大的」會先浮到水面頂端。

氣泡排序就是這樣的概念——每輪從清單左端出發，相鄰兩個比一比，左邊比右邊大就互換，這樣最大的值會像氣泡一樣，一格一格往右「浮」，最終到達它應該在的位置。

一句話記住：每輪從左到右，相鄰兩兩比較，左大右小就交換。每輪結束，目前最大的那個值一定已浮到最右邊（鎖定），下一輪不用再碰它。

和另外兩種排序的比較

三種排序，三種策略

  • 選擇排序：先掃完整個未排序區，找出最小值，才做一次交換。
  • 插入排序：拿起一個元素，往左找到正確位置後插入，已排序區在左側增長。
  • 氣泡排序：從左到右，相鄰兩兩比較，隨時發現左大右小就立刻交換，最大值自然浮到右端，已排序區從右側增長。
二、核心動作與提前停止
每輪的動作
  1. 從清單的第 1 項開始，依序比較「第 j 項」和「第 j＋1 項」。
  2. 若「第 j 項」大於「第 j＋1 項」→ 互換兩者（需要 temp）。
  3. j 往右移一格，重複，直到比完本輪最後一對。
  4. 本輪結束，最右邊那個位置的值已確定（鎖定），下一輪不再碰它。

提前停止（最佳化）

如果某一輪從頭比到尾，一次交換都沒有發生，代表清單已經完全排好，不需要再繼續跑下去。

旗標變數：「已交換」

每輪開始前把 已交換 設為 0（代表「這輪還沒換過」）；每次真正交換時把 已交換 設為 1。輪結束後，若 已交換 = 0 → 排序完成，立刻停止外層迴圈。

提前停止的威力：若資料已幾乎排好，氣泡排序可以非常快結束。例如清單只有最後兩個元素對調，第一輪修正後，第二輪偵測到 0 次交換就立刻停止，比選擇排序少跑很多輪。

四、流程步驟與比較次數
Scratch 讀法：下面用「將…設為」「清單的第（　）項」「重複執行直到」來說明。第（j）項與第（j）＋（1）項是相鄰兩格；內層「要比到哪一項為止」請先用變數「上限」存「（清單「資料」的項目數）－（i）」，再用「j 從 1 一路做到上限」最好接。
氣泡排序流程（由小到大，含提前停止 · 積木口語）

想像成「每一輪把比較大的數字像氣泡一樣往右推；若某一輪完全沒交換，代表已經排好，就可以提早結束」：

  1. 外層（最多 n－1 輪）：將「i」從 1 開始；每一輪代表「最右邊已經鎖定的區塊再多固定一格」。
  2. 每一輪外層一開始：將「已交換」設為 0（先假設這輪還沒換過）。
  3. 先算出內層要比到哪裡：將「上限」設為「（清單「資料」的項目數）－（i）」（避免一直比到已經排好的最右端）。
  4. 內層（相鄰兩兩比）：將「j」設為 1，然後「重複執行直到（j）>（上限）」：若「（清單「資料」的第（j）項）>（清單「資料」的第（（j）＋（1））項）」，就用「temp」做三步驟交換，並把「已交換」設為 1；最後「將 j 增加 1」。
  5. 內層整輪結束後（提前停止）：如果「（已交換）= 0」，代表這一輪完全沒換過 → 清單已排好 → 可用「停止全部」或改寫外層「重複直到」條件來結束整個排序。
為什麼內層只到「項目數－i」？第 i 輪外層結束後，最右邊 i 個位置已經是「本輪最大的一批」排好了；下一輪就不需要再拿它們出來互相比，所以「上限」會愈來愈短。

比較次數：最好 vs 最壞

情況說明輪數比較總次數
最好情況 資料已由小到大排好 1 輪（偵測到 0 次交換，立刻停） n－1 次
最壞情況 資料由大到小（完全倒序） n－1 輪（每輪都有交換） n×(n－1)÷2 次

三種排序的提前停止能力比較

  • 選擇排序：不能提前停止外層迴圈，永遠要跑完 n－1 輪。
  • 插入排序：內層可以提前停（key 找到位置就停），但外層仍要跑 n－1 輪。
  • 氣泡排序：外層可以提前停（一整輪「已交換仍為 0」就結束）——是三種中唯一能提前結束整個排序的。

Scratch 的內層範圍：請先把「上限 ←（清單「資料」的項目數）－（i）」算出來，再讓「j」從 1 做到「上限」，避免每一輪都比到已經鎖定的右端區塊。
提前停止的接法：上面示範的是「內層整輪跑完 → 如果（已交換）= 0 → 停止全部」；你也可以改成外層用「重複直到」一次包進更多條件，以課堂教學為準。請勿把「（已交換）= 0」直接寫在外層一開始就判斷，否則內層還沒比就會誤判結束。

五種演算法總整理

演算法 目的 需先排序？ 最好比較次數 最壞比較次數 外層能提前停？
線性搜尋 找目標 不需要 1 次 n 次 ✓（找到就停）
二元搜尋 快速找目標 要 1 次 約 log₂n 次 ✓（找到就停）
選擇排序 排序 — n×(n－1)÷2 n×(n－1)÷2 ✗（不能）
插入排序 排序 — n－1 次 n×(n－1)÷2 △（內層可以）
氣泡排序 排序 — n－1 次 n×(n－1)÷2 ✓（外層可以）
學習路徑回顧：線性搜尋 → 選擇排序 → 插入排序 → 氣泡排序 → 二元搜尋。先學會用排序把資料整理好，再用二元搜尋快速查找，這就是演算法入門的完整拼圖！
八、延伸思考（選做）
  • 若改成「由大到小」排序，流程裡哪個比較符號要改？改成什麼？
  • 清單 [1, 3, 2, 4, 5] 用氣泡排序，第幾輪會觸發提前停止？總共比了幾次？
  • 氣泡排序和插入排序在最好情況下都只需要 n－1 次比較，但「最好情況」的含義一樣嗎？（提示：各自是什麼樣的輸入才能達到最好情況？）
  • 選擇排序、插入排序、氣泡排序在最壞情況下比較次數都是 n×(n－1)÷2。如果要排 100 個數字，你會選哪一種？說說你的理由。
